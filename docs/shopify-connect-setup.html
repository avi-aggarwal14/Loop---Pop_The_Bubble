Synapse
Connect a Shopify store — end-to-end setup & test guide
What this is for. The store you create here is a test fixture, not part of the product. Its only job is to let you click through the real connect flow once and confirm every link works. In production, each real founder connects their own store and the app reads their data — nothing about your store is hardcoded anywhere. The dashboard always reads whichever store the current visitor connected (via a signed session cookie).

Does it work on a completely empty store?

Connecting: yes — OAuth doesn't care whether the store has data. The connection succeeds either way.
Getting a customized answer: needs a little data. With zero orders the data pull comes back empty and the engine falls back to the built-in demo example, so you'd see a generic answer instead of your store. Real founders never hit this — their stores already have months of orders. For your test store, add ~5 fake orders (see the last section) so the real path actually fires.

1. Create the store

A Shopify development store via Shopify Partners is ideal — free, no billing. If you created it through partners.shopify.com you already have the Partners account needed for step 3.

2. Shopify Partners account

partners.shopify.com — free. Your dev store lives under it.

3. Create the app & get the keys

In the Partners dashboard:

  • Apps → Create app → Create app manually → name it "Synapse".
  • Open the app → Client credentials (a.k.a. API keys). Copy:
    • Client ID → this is SHOPIFY_API_KEY
    • Client secret → this is SHOPIFY_API_SECRET

4. Set the App URL + redirect (must match exactly)

In the app → Configuration:

  • App URL: https://synapse-acceleration.vercel.app
  • Allowed redirection URL(s): https://synapse-acceleration.vercel.app/api/auth/shopify/callback

The redirect string must be character-for-character exact, or Shopify rejects the callback.

5. Add env vars to Vercel — then redeploy

This is the one part only you can do. These go on the Vercel project that serves synapse-acceleration.vercel.app (your teammate's project), not just your local machine.

In that project's Settings → Environment Variables, add:

NameValue
SHOPIFY_API_KEYthe Client ID from step 3
SHOPIFY_API_SECRETthe Client secret from step 3
APP_URLhttps://synapse-acceleration.vercel.app

Scopes default to read_orders,read_customers,read_products,read_reports already — you don't need to set SHOPIFY_SCOPES unless you want to change them. Then Redeploy (env vars only take effect on a new deploy).

6. Test the flow

  1. Go to https://synapse-acceleration.vercel.app/dashboard
  2. In the Shopify card, type your-store.myshopify.com → Connect
  3. Shopify shows an approve screen (proves keys + redirect are right) → approve
  4. You land back on /dashboard?connected=shopify; the card shows ● Connected with your shop name
  5. Ask Synapse is now live against your real store data

Until those env vars are set + deployed, the Shopify card shows "Connection coming soon" instead of the connect box — that's the graceful fallback, not a bug.

To actually see real numbers — add a little data

Quickest way on a dev store:

  1. Add 2–3 products (Products → Add product).
  2. Place a few test orders with the Bogus Gateway: Settings → Payments → add "(for testing) Bogus Gateway", then order through your storefront using card number 1 (2/3 simulate failures).

Even ~5 orders across the last week or two is enough for the Ask to reason about your store (revenue, top product, channel mix) instead of falling back to the demo example.

The mental model.
Production reality: a user connects their store → their data → their answer. Already built.
Your test store: the one store you can legitimately connect to confirm that reality works before the event — and it needs a sprinkle of data only because it starts empty.
